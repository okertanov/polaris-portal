Development process

All dVITA token holders will have the right to vote and participate in dVITA’s technical governance and development process by electing qualified dVITA holders to become consensus node operators and members of dVITA’s technical governing body. Active participation in dVITA’s governance will entitle token holders to earn incentive rewards in the form of DVG.

Local dVITA node development setup

Requirements
  • Dev account in https://gitlab.team11.lv
  • Windows, MacOS or Linux developer station
  • Docker & Docker Compose
To check if Docker and Docker Compose are installed, use commands: 
                                        docker --version
                                    
                                        docker-compose --version
  • .NET6 SDK & Runtime
To check .NET version, use command: 
                                        dotnet --info
  • GIT Client
To check if GIT is installed, use command: 
                                        git --version
  • GNU Make
To check if GNU Make is installed, use command: 
                                        make --version

Development environment setup 

                                    # Clone all following repositories inside one directorly like:

                                            mkdir -p ~/projects/nxa-project

                                            cd nxa-project

                                            git clone https://gitlab.team11.lv/nxa/nxa-node-infra.git

                                            git clone https://gitlab.team11.lv/nxa/mirrors/neo-vm.git

                                            git clone https://gitlab.team11.lv/nxa/mirrors/neo.git

                                            git clone https://gitlab.team11.lv/nxa/mirrors/neo-modules.git

                                            git clone https://gitlab.team11.lv/nxa/mirrors/neo-node.git

                                            git clone https://gitlab.team11.lv/nxa/nxa-modules.git

                                            git clone https://gitlab.team11.lv/nxa/nxa-dev-solution.git

                                            

                                            # Pre-build modules

                                            cd neo-modules

                                            make build

                                            

                                            # Build node

                                            cd ../neo-node/neo-cli

                                            make build

                                            

                                            # Copy binary deps

                                            cp ../nxa-dev-solution/libleveldb.dll ./

                                            

                                            # Install default modules that have been build above

                                            mkdir ./dist/Plugins

                                            cp -a ../../neo-modules/dist/* dist/Plugins

                                            

                                            # Remove 'Storage' and 'Logger' sections

                                            vim dist/config.json

                                            

                                            # Run it locally

                                            dotnet dist/neo-cli.dll

How to perform Blockchain transactions

1. Install wallet plugin and generate new address [A]
  • Git clone https://gitlab.team11.lv/nxa/mirrors/neoline.git
  • Run npm install to install dependencies.
  • Run npm run build to build release assets for chrome extension debug.
  • Open google chrome manage extensions page chrome://extensions/
  • Turn on developer mode
  • Drag & drop neoline dist folder
  • Create wallet through installed Chrome plugin
2. Run node instance and create wallet [B] through node CLI
  • Run node following instructions described on How to run dVITA node instances
  • Inside node CLI create wallet:
                                create wallet wallet.json
    and input a password for the wallet
  • Returned values: 
                                
                                    Address: NQD93fVJqr5YLm7jYZaVnPQbsXPc7jFzpv

                                    Pubkey: 025999b89e08b0bcaf4dbb5a82180dba7ffbd1d51000c71c7a2103ac35b2968d15

                                    ScriptHash: 0x458de6187b6347b9bbc905b605c99f5c3cce1f2f

                                    DVITA hash: 0xb34e1025391e953a918231df11478ec21b039e5f

                                    DVG hash:   0xd2a4cff31913016155e38e474a2c06d08be276cf
3. Deposit DVITA and DVG to [A] via https://foundation.dvita.com/
4. Send [A] → [B] DVG and DVITAs
    Using wallet plugin, transfer chosen amount to NQD93fVJqr5YLm7jYZaVnPQbsXPc7jFzpv
5. Check [B] balance
    a. Using list asset command
  • In CLI type list asset
    • b. Using balanceof command
  • For DVITA balance balanceof 0xb34e1025391e953a918231df11478ec21b039e5f NQD93fVJqr5YLm7jYZaVnPQbsXPc7jFzpv
    • or balanceof DVITA NQD93fVJqr5YLm7jYZaVnPQbsXPc7jFzpv
  • For DVG balance balanceof 0xd2a4cff31913016155e38e474a2c06d08be276cf NQD93fVJqr5YLm7jYZaVnPQbsXPc7jFzpv
    • c. Through smart contract invocation

  • Account details

  • Input wallet name, Private Key and password
    For [B]

  • open wallet wallet.json
  • Input password
  • import key <Private key>
7. Compile and deploy token through CLI
  • Create Smart Contract
  • Compile it using PUT https://caas.testnet.dvita.com/api/Compiler or Neo.Compiler.CSharp
  • Create sc directory
  • TokenName.nef and TokenName.manifest.json into sc directory
  • Copy this directory inside docker container docker cp ./sc dvita-testnet-node:sc
  • Run deploy command inside node CLI deploy ../sc/TokenName.nef ../sc/TokenName.manifest.json
    • and as a response get Contract hash: 0x1c6c0fb668ca7282601c7891416dcdaaf2e41ae1
  • Create wallet through installed Chrome plugin
8. Interact with this contract
